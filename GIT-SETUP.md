# GitHub setup — XRPName MCP Server

One-time steps to put this project on GitHub as a **private** repo. Run these in
a terminal (PowerShell or Git Bash) on your own machine — not in the sandbox.

> Why your machine: the project lives in a OneDrive folder; the safest place to
> commit the real, fully-synced files is locally.

## 1. Initialise the repo + identity

```bash
cd C:\Users\PC\OneDrive\Develop\VibeCode\xrpname-mcp-server

git init -b main
git config user.name  "Redstone"
git config user.email "hubsocialvn@gmail.com"
```

## 2. Generate the lockfile (required by CI)

`.github/workflows/ci.yml` runs `npm ci`, which **needs `package-lock.json`**.
Generate it once and let it be committed:

```bash
npm install        # creates package-lock.json
```

## 3. First commit

```bash
git add .
git commit -m "Initial commit: XRPName MCP Server (Bước 1 — 4 read tools)"
```

## 4. Create the GitHub repo and push

Pick **one** of the two options.

### Option A — GitHub website (no extra tools)

1. Go to <https://github.com/new>
2. Repository name: `xrpname-mcp-server`
3. Visibility: **Private**
4. Do **not** tick "Add a README / .gitignore / license" (the repo already has them)
5. Click **Create repository**, then run:

```bash
git remote add origin https://github.com/XRPDomains/xrpname-mcp-server.git
git push -u origin main
```

> Repo already created at: https://github.com/XRPDomains/xrpname-mcp-server

**If you created the repo WITH a README/license** (so it isn't empty), the first
push will be rejected. Reconcile the histories once, then push:

```bash
git pull origin main --allow-unrelated-histories
git push -u origin main
```

### Option B — GitHub CLI (`gh`)

If you install GitHub CLI (<https://cli.github.com>), one command does it all:

```bash
gh auth login            # first time only
gh repo create xrpname-mcp-server --private --source=. --remote=origin --push
```

## Notes

- **Secrets:** there is no `.env` in the repo — only `.env.example`. Never commit
  a real `.env`; `.gitignore` already blocks it.
- **Build artifacts** (`bundle/`, `*.mcpb`) are git-ignored because they are
  regenerated by `npm run pack`. To ship a build, attach the `.mcpb` to a GitHub
  **Release** instead. (Remove those two lines from `.gitignore` if you'd rather
  track them.)
- **CI:** on every push to `main` and every PR, GitHub Actions runs typecheck +
  lint + tests. The green check confirms the build is healthy.
- **OneDrive:** having `.git` inside a OneDrive folder works, but avoid editing
  the repo on two machines at once to prevent sync conflicts in `.git/`.
